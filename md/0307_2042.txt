› AdminPostBulkDeleteRequest 의 프로퍼티는 List<Long> postIds 이거 하나인데 굳이 해당 DTO를 생성한 이유를 답해주세요.


• 좋은 지적입니다. 현재 요구사항만 보면 List<Long> 직접 받는 것도 충분합니다.

  제가 AdminPostBulkDeleteRequest를 만든 이유는 4가지입니다.

  1. API 계약 명확화

  - 요청 바디가 {"postIds":[...]} 형태라는 걸 메서드 시그니처에서 바로 드러내기 위해서입니다.
  - 단순 배열([...])보다 문서/클라이언트 구현 시 오해가 적습니다.

  2. 확장성

  - 나중에 reason, requestedBy, hardDelete 같은 필드가 추가돼도 메서드 시그니처 변경 없이 DTO만 확장하면 됩니다.

  3. 검증/바인딩 포인트 확보

  - @Valid, @NotEmpty 같은 검증을 DTO 필드에 붙이기 쉽습니다.
  - List<Long> 직접 바인딩보다 에러 메시지/검증 관리가 수월합니다.

  4. 일관성

  - 이 프로젝트의 다른 요청들도 DTO 기반이 많아서 컨트롤러 스타일을 맞추려는 의도였습니다.

  원하시면 간소화해서 @RequestBody List<Long> postIds로 바로 받도록 바꿔도 됩니다.
