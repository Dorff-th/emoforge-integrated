3. Security 관련 import 또는 bean 설정에 영향이 있는 Java 파일이 있는지 확인하라.
4. 각 변경에 대한 예상 리스크를 설명하라.

현재 프로젝트에서 MSA 잔재로 보이는 Feign(OpenFeign) 사용 여부를 분석하라.

분석 대상:
- spring-cloud-starter-openfeign dependency
- @FeignClient 사용 코드
- Feign 관련 configuration
- Feign interceptor
- Feign DTO 또는 legacy client 코드

요청 작업:

1. 프로젝트 전체에서 @FeignClient 사용 위치를 모두 찾는다.
2. 각 FeignClient가 호출하는 대상 서비스(auth-service, attachment-service 등)를 정리한다.
3. 현재 Monolith 구조에서 동일 기능을 수행할 수 있는 내부 Service 호출로 대체 가능한지 분석한다.
4. 실제로 사용 중인 코드와 "레거시로 추정되는 코드"를 구분한다.
5. OpenFeign dependency를 제거할 경우 영향을 받는 파일 목록을 작성한다.
6. 안전하게 제거하기 위한 단계별 migration plan을 제시한다.
