Context:
This project uses a shared AppHeader with context-specific center components
(PUBLIC, POST, USER, DIARY).

Current issue:

because the center section (PostHeader search input) occupies too much width.

Requirements:
1. Keep the desktop layout unchanged.
2. On mobile screens:

   - hide ThemeToggle
   - ensure ProfileMenu never shrinks or overflows
3. Use Tailwind responsive utilities.
4. Do not break existing header context logic.

Please modify:
- AppHeader.tsx
- AppHeaderRight.tsx
- PostHeader.tsx

Also explain the layout changes briefly.
