# 2️⃣ 발생한 고민 / 이슈

### (1) 탈퇴 회원만 삭제 가능하도록 할 것인가

관리자 실수 방지 필요.

---

### (2) 탈퇴일 표시 UX

단순 날짜만 표시 vs 경과일 표시

예

2026-03-12 (1일)

---

### (3) 마지막 로그인 저장 방식

로그인 기록을 전부 DB에 쌓아야 하는지 고민

예

login_history 테이블

→ DB 증가 및 서버 부담 우려.

---

### (4) 관리자 계정도 lastLoginAt 필요 여부

---

### (5) 테이블 UI 정렬

- `-` 표시 가운데 정렬
    
- 전체 td 정렬 방식 고민
    

---

# 3️⃣ 해결 방법

## (1) 탈퇴 회원만 삭제 가능

UI

disabled={!m.deleted}

백엔드

deleted = true 인 경우만 삭제 허용

→ **이중 보호 구조 적용**

---

## (2) 탈퇴일 + 경과일 표시

예

2026-03-12 (1일)

React util

formatWithdrawDate()

---

## (3) 마지막 로그인 구현 방식

로그 테이블 없이

member.last_login_at

컬럼 추가

ALTER TABLE member  
ADD COLUMN last_login_at DATETIME NULL;

로그인 성공 시

member.setLastLoginAt(LocalDateTime.now());

---

## (4) 관리자 계정도 lastLoginAt 기록

관리자도 동일한 `member` 테이블 사용.

보안 및 계정 관리 측면에서 필요.

---

## (5) 테이블 정렬 방식 개선

UUID / 닉네임 → left  
상태 / 날짜 / 버튼 → center

`tbody` 기준 center 정렬 적용.
